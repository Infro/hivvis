What's new in Entrez Utilities Web Service

Updated: December 14, 2010

Changes in version 2.0 (12/14/2010)

 

·         PubMed WSDL is generated for new DTD 2011 version. The PubmedArticleSet array now includes items of PubmedArticle and PubmedBookArticle types. Please see example of eFetch/PubMed call: Using WebEnv & QueryKey example

·         A workaround for efetch_pmc known issue is added for Java Axis2. See Using xmlbeans binding paragraph.

Previous changes


The biggest change is that the eFetch method has been removed from the eUtils.wsdl file. Now developer should use corresponding WSDL file to fetch records from a database, for example, efetch_pubmed.wsdl to fetch records from PubMed. The other changes are:

  • All schemas have been regenerated based on current DTDs (April 17, 2009)
  • Fixed binding problem in WSDL files *
  • Added EPost utility support
  • mixed="true" has been eliminated from eSummary schema;
  • linkname property in ELinkRequest element
  • eUtils_lite.wsdl, efetch.wsdl, and efetch_lit.wdl do not exist anymore;
  • Each new version of the Web Service will be placed into the corresponding directory. For example, version 2.0 is located in ../eutils/soap/v2.0/ directory.

 

Version 2.0 is not fully compatible with the previous version. Developers should regenerate source code with the new WSDL. The following should be done to update the project:

  • use eUtils.wsdl instead of eUtils_lite.wsdl; eUtils_lite.wsdl does not exist anymore
  • use eUtils_MS.wsdl if there are _MS methods calls in the source code
  • use efetch_<db-name>.wsdl instead of efetch.wsdl and efetch_lit.wsdl; for example efetch_gene.wsdl to fetch records from Gene database. If you need to fetch records from multiple databases then add a Web Reference per each database in MSVS. For Axis2 run WSDL2Java for each database you need and include the package to project. Each efetch_<db-name>.wsdl file has different namespace to prevent name conflicts.

 

(*) Methods with _MS suffix are moved to eUtils_MS.wsdl file, which should be used in MS Visual Studio only. Those methods have different style of input parameters, i.e. a list of parameters in method's definition instead of one object.
