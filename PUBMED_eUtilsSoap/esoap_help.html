NCBI Entrez Utilities Web Service

Updated: December 14, 2010

 

The NCBI Entrez Utilities Web Service enables developers to access Entrez Utilities via the Simple Object Access Protocol (SOAP).

The Web Service has been tested with:

  • C# and Visual Basic in MS Visual Studio 2008
  • Java (Apache Axis2 version 1.5.2)  

The Web Service also should work with the older versions of Axis (Axis for Java ver. 1.4) and MS Visual Studio (MSVS 2005).

Developers can also use other SOAP libraries and tools to access NCBI Entrez Utilities Web Service, for example MS SOAP Toolkit 3.0 and MS Word Visual Basic.

Some features might be not working with clients that we didn’t test.

Version 2.0 (current)

 

What's new in version 2.0

The NCBI Entrez Utilities Web Service  v2.0 provides access to EGQuery, ESummary, EInfo, ELink, ESearch, ESpell, EPost and EFetch utilities.

To create a SOAP client application developer should use one or more of the following WSDLs directly from our web site or download WSDL/XSD files from our FTP site:

  • http://eutils.ncbi.nlm.nih.gov/soap/v2.0/eutils.wsdl, which includes all utilities but EFetch.
  • WSDL to fetch records from a corresponding database:

o    Pubmed:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_pubmed.wsdl

o    PMC:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_pmc.wsdl

o    NlmCatalog:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_nlmc.wsdl

o    Journals:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_journals.wsdl

o    OMIM:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_omim.wsdl

o    Taxonomy:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_taxon.wsdl

o    SNP:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_snp.wsdl

o    Gene:

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_gene.wsdl

o    Sequences :

http://eutils.ncbi.nlm.nih.gov/soap/v2.0/efetch_seq.wsdl

    Sequences are protein, nuccore, nucgss, nucest, genome, and popset databases.

The input parameters for the SOAP version of the Entrez Utilities are the same as described in the documentation for each individual utility except no retmode parameter is available as the SOAP implementation only returns XML.

Known issues

  • MSVS 2008 C#/VB application fails when eUtils SOAP response size exceeds the default MaxReceivedMessageSize value (65535 bytes). Increase all maxReceivedMessageSize and maxBufferSize parameters in the app.config file to the same value, for example, 2000000.
  • MS Visual Studio 2008 fails to generate a source code for efetch_pmc.wsdl because it includes MathML schema which has a circular group references.
  • WSDL2Java from Axis2 package also fails with efetch_pmc.wsdl and default adb binding. Use –d xmlbeans binding to get valid java source code for efetch_pmc.wsdl. efetch_pmc_beans.jar is available for download on our FTP site. Also, see Using xmlbeans binding example.
  • WSDL2Java generates for efetch_gene.xsd a source code which could not be compiled. This can be fixed if you add some prefix to local variables names in Value_type47 declaration. For example, add "x" as a prefix, so _fun variable become x_fun and fun variable become xfun, etc. There 34 variable names to change: _fun, _inv, _mam, _org, _phg, _pln, _pri, _pro, _rod, _syn, _una, _vrl, _vrt, _pat, _est, _sts, _other, fun, inv, mam, org, phg, pln, pri, pro, rod, syn, una, vrl, vrt, pat, est, sts, other.
